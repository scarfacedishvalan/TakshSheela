# Scenarios — prob-001

Each subdirectory here is a fault-injected variant of the canonical codebase.

Naming convention: `<scenario-id>_<fault-family>_<subtype>/`

Contents of each scenario folder:
- `fault_scenario.yaml` — injected fault metadata
- `observability/` — logs, metrics, alerts, traces
- `assessment_package.yaml` — candidate-facing incident materials
